Description

Below information must be included in the monthly report: 1. Head Contract Summary a) Head Contract Value b) Variations Approved to Date c) Revised Contract Sum d) Tender Margin e) Amendment to tender margin (brief explanation of why + key figures) f) Final Tender Margin g) Construction Budget 2. Financial Details (table) For each item/line in the table: a) Description b) This Month value c) Comments (only where needed to explain movement/variance) 3. Variation Report (high-level totals) a) Approved Variations (total) b) Pending Variations (total) c) Declined Variations (total) d) Internal Variations (total) 4. Margin Forecast a) Current Construction Budget b) Committed to date c) Forecast to complete d) Projected Letting Gains e) Tender Margin f) Forecast Total Margin 5. Progress Claim Summary For each claim: a. Claim number b. Claim amount c. Status (e.g., PAID) d. Total claimed to date 6. Approved Variations Summary (detailed list) For each approved variation line item (as shown in the summary table): a. Variation reference / title b. Value c. Status (e.g., SUBMITTED / INVOICED / PAID where applicable) d. Total approved variations 7. Declined Variations Summary (detailed list) For each declined variation line item: a. Variation reference / title b. Value c. Reason / brief note (if the sheet has a comments/notes area for it) d. Total declined variations 8. Internal Variations Summary (detailed list) For each internal variation line item: a) Reference / title b) Value c) Comment on why it’s internal + impact d) Total internal variations 9. Program Update a) Current program status vs baseline b) Key delays / drivers / recovery actions c) Commentary note (your template note says to attach the program showing the “current date” line) 10. Procurement a. Procurement status summary (what’s procured / what’s outstanding) b. Any long-lead risks and required actions 11. Risks a. Key current risks (commercial / program / delivery) b. Mitigation actions c. Any budget impacts or approvals needed 12. Incidents a. Incidents summary (if none, state “Nil”) b. Corrective actions / close-out status 13. General Notes a. Audits (outcomes / actions / close-out) b. Client instructions / key directions c. Any other notable project notes for the month
